Build a production-grade web app called "Maintenance Task & QC Management System" with an advanced interactive equipment selector focused on gas turbine maintenance.

TECH STACK:
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui
- Prisma ORM
- PostgreSQL
- NextAuth (email login)
- react-signature-canvas
- Zustand for state (lightweight)
- File upload ready for S3/R2

----------------------------------------
CORE ROLES:
1) Engineer / Supervisor / Site Manager
- create tasks
- assign tasks
- set estimated time and deadline
- review submitted work
- approve or reject QC

2) Technician
- view assigned tasks
- start / pause / resume tasks
- add execution notes
- upload photos
- sign electronically
- submit for QC

----------------------------------------
TASK STATUS FLOW:
Draft → Assigned → In Progress → Paused → Submitted → Under QC → Approved / Rejected → Overdue

----------------------------------------
MAIN FEATURE: INTERACTIVE TURBINE SELECTOR

Create a structured + visual selector for equipment location.

STEP 1 — SECTION SELECTION:
Display gas turbine sections:
- Compressor
- Mid Frame
- Turbine
- Turbine Exit Cylinder

UI:
- Show turbine image at top (static for now)
- Below it: selectable cards/buttons
- Highlight selected section

STEP 2 — IF "TURBINE" SELECTED:
Show turbine stages:

- Stage 1 (Blade count: 80–100)
- Stage 2 (Blade count: 70–90)
- Stage 3 (Blade count: 60–80)
- Stage 4 (Blade count: 50–70)

UI:
- Cards or horizontal selector
- When selected → show info panel:
  "Approximate blade count: X–Y"

STEP 3 — COMPONENT SELECTION:
After selecting stage, show:

- Rotor Blade
- Stator Vane
- Seal
- Casing
- Shaft
